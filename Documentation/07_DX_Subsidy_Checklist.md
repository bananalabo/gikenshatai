# DX補助金 チェックリスト（要件確認メモ）

## 1. 目的
DX補助金の対象になり得るかを判断するため、要件（特に「パッケージ化」「販売」等）を確認する。

## 2. 確認事項（最優先）
- 「開発物」をパッケージ化して販売することが必須か
- 対象となるのは自社内利用のみか、第三者提供/販売も対象か
- パッケージ化の定義（利用料体系、導入形態、ライセンス形態）
- MVP段階でも対象可否（開発完了時期の要件）
- 対象業種（建設業/自動車修理など）の扱い

## 3. 技術・運用面での確認
- オンラインサービス形態（Web）で問題ないか
- 写真ストレージ/サーバー構成はどう扱われるか（補助対象経費）
- ランニングコストは補助対象か（または対象外か）
- セキュリティ要件（アクセス制御、ログ等）

## 4. 必要書類（準備しておくもの）
- 企画書（本フォルダの `01_Platform_Proposal.md`）
- PRD/MVP要件定義（`02_MVP_PRD.md`）
- 見積仕様/概算見積（`06_Dev_Estimation_Spec.md`）
- 既存業務の課題資料（`03_AsIsToBe_Workflow.md`）

## 5. 次アクション
- 補助金窓口に上記「最優先」項目を質問する
- 回答内容をこのドキュメントに追記し、対象可否の結論を残す
