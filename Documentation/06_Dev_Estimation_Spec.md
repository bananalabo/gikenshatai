# 開発見積依頼仕様（MVP）

## 1. 目的
車両修理のB2B案件を、`案件ID`単位で「写真・PDF・ステータス」を一元管理するWebプラットフォーム（MVP）を開発し、LINE/メール多段転送を削減する。

## 2. MVPスコープ（機能）
### 2.1 ログイン/アカウント
- 管理者（元請け）と利用者（ディーラー/法人）を分ける
- ロールごとにアクセス制御（他社案件を見えないようにする）

### 2.2 案件管理
- 案件作成（利用者）
- 案件一覧（管理者/利用者それぞれの閲覧範囲で）
- 案件詳細（案件ID単位）

### 2.3 添付（写真・PDF）
- 写真：複数枚アップロード、サムネイル一覧、クリック拡大/プレビュー
- PDF：見積/納品書等の所定欄へ添付（最小UIでOK）
- 添付の種類（見積/納品/途中写真/その他）を紐付け可能にする

### 2.4 ステータス更新（最小）
- ステータスは初期例：受付/確認中/見積済/作業中/納品済/完了
- 管理者が更新（MVPでは利用者更新は限定 or 非対応でも可）

### 2.5 モバイル対応
- スマホで写真撮影→アップロード→登録が可能なUI

## 3. 画面要件（最小セット）
- ログイン
- 管理者：案件一覧、案件詳細（1画面寄り）
- 利用者：新規案件作成、案件詳細（アップロード中心）

## 4. ストレージ前提
- 写真品質/UXを優先し、MVPでは自前サーバー基盤での実装を前提とする
- ただし月額 `3〜5万円` 目標に収めるため、容量/トラフィックを含めて見積に含める

## 5. データ/権限設計の前提
- `User` は `Organization` に紐づく
- `Case` も `Organization` に紐づく（MVP：利用者は自組織案件のみ）

## 6. スケジュール案（たたき台）
※見積時に提案してもらう前提（開発会社の見立てにより調整）
- Phase 1：要件確定・画面設計（1〜2週間）
- Phase 2：MVP実装（2〜4週間）
- Phase 3：連携/権限/アップロード品質の調整（1〜2週間）
- Phase 4：デモ・受け入れ・修正（1週間）

## 7. 見積で確認したい項目
- 開発費（初期）総額
- ランニング費（サーバー/ストレージ/運用）
- 画像サイズ・保存形式・バックアップ方針
- 権限/セキュリティの実装方法
- 管理画面の柔軟性（ステータス追加など）
- テスト方法と受け入れ条件（最低限：アップロード〜閲覧の正確性）

## 8. 成果物（納品物）
- MVP Webアプリ（管理者/利用者）
- 管理者向け操作手順書（最低限）
- 運用時のルール（削除/編集、誤添付時の扱い等）
