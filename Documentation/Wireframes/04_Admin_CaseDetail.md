# Wireframe: 案件詳細（1ページ：Admin版）

## 目的
`案件詳細`は「1ページで完結」させる（Admin/Userでレイアウトは同じ）。  
差分は `Adminだけ表示/操作できる領域（ステータス更新）`のみ。

## レイアウト（案件ヘッダ＋所定欄＋途中写真＋Admin-only更新）
```
┌─────────────────────────────────────┐
│ 上部：案件詳細（上部ナビ：案件一覧へ戻る）│
├─────────────────────────────────────┤
│ 案件ID：000123                        │
│ 関連先：Honda（例）                   │
│ ステータス：見積済                    │
├─────────────────────────────────────┤
│ 所定欄（PDFなど）                     │
│ [ 見積PDF ]  [アップロード/差し替え ] │
│ [ 納品書PDF ] [アップロード/差し替え] │
│ （Admin-only：入力UI。Userのアップロードは反映される） │
├─────────────────────────────────────┤
│ 途中写真（複数）                     │
│ [サムネイル] / [プレビュー]         │
│ （Admin-only：写真追加入力は非表示）  │
├─────────────────────────────────────┤
│ ステータス更新（Admin-only）           │
│ [受付/確認中/見積済/作業中/納品済/完了] │
│ [更新する]                            │
└─────────────────────────────────────┘
```

## 注記（仕様）
- 添付の種類はMVPでは固定カテゴリ（見積/納品/途中/その他）で迷わせない
- User側のアップロード/保存内容がこの画面に反映される前提
- 写真追加入力はAdminでは隠す（MVP事故防止）
