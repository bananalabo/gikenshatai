# Wireframe: 案件詳細（1ページ：User版）

## 目的
`案件詳細`は「1ページで完結」させる（Admin/Userでレイアウトは同じ）。  
差分は `Userが入力できる領域（所定欄アップロード＋写真追加入力）`のみ。

## レイアウト（案件ヘッダ＋所定欄＋途中写真＋User-only入力）
```
┌─────────────────────────────────────┐
│ 上部：案件詳細（上部ナビ：案件一覧へ戻る）│
├─────────────────────────────────────┤
│ 案件ID：000123                        │
│ 関連先：Honda（例）                   │
│ ステータス：受付/確認中            │
├─────────────────────────────────────┤
│ 所定欄（PDFなど）                     │
│ 見積PDF： [アップロード] [保存]     │
│ 納品書PDF： [アップロード] [保存]   │
│ （User-only：入力UI。Admin-onlyは非表示）│
├─────────────────────────────────────┤
│ 途中写真（複数）                     │
│ [サムネイル] / [プレビュー]        │
│ 写真追加： [写真を追加]（スマホ想定）│
└─────────────────────────────────────┘
```

## 重要なUX要件（MVP）
- スマホで迷わないように「所定欄」を固定（欄ごとにアップロード）
- 保存ボタンと入力欄の距離を短くする（ミスを減らす）
- ステータス更新は事故防止でUserでは非表示（Adminのみ）
