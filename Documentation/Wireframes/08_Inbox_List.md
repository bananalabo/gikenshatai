# Wireframe: インボックス一覧（Inbox List）

## 目的
案件に関する依頼・確認・差し替えなどのメッセージを、1つの入口（インボックス）に集約する。

## レイアウト（SaaS左ナビ＋メイン）
```
┌───────────────────────────────┐
│ 左ナビ                           │
│ - 案件一覧   │ - 新規依頼（Userのみ）│
│ - インボックス（active）         │
└───────────────────────────────┘

┌────────────────────────────────┐
│ インボックス                     │
├────────────────────────────────┤
│ [ 未読 ] [ 全て ]                │
├────────────────────────────────┤
│ メッセージ一覧（テーブル/カード） │
│   - ステータス（未読/既読）        │
│   - 件名                           │
│   - 案件ID                          │
│   - [開く]（スレッドへ）            │
└────────────────────────────────┘
```

## 注記（仕様）
- User/Adminで同じ画面。差分はアクションの可否（必要なら）
- MVPではダミー表示でOK（後で実データに差し替え）
