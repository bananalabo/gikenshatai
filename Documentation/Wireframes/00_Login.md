# Wireframe: ログイン画面（Login）

## 想定ロール
- 管理者：元請け
- 利用者：ディーラー/法人

## レイアウト（1画面完結）
```
┌─────────────────────────────────────┐
│ 企業ロゴ / アプリ名                 │
├─────────────────────────────────────┤
│ [説明文]                             │
│ 「案件管理プラットフォームへ」     │
├─────────────────────────────────────┤
│ ID（メール/ユーザー名）           │
│ [入力欄___________________________] │
│ パスワード                          │
│ [入力欄___________________________] │
│                                     │
│ [ログイン]                           │
├─────────────────────────────────────┤
│ 連絡先 / 初期パス発行（必要なら）    │
└─────────────────────────────────────┘
```

## ボタン/アクション
- `ログイン`

## 状態/エラー
- 入力エラー（必須未入力、形式不正）
- 認証失敗（メッセージは抽象的に：例「IDまたはパスワードが違います」）

## 注記（仕様）
- ログイン後に `Organization` と `Role` に応じて遷移先（案件一覧）を変える
