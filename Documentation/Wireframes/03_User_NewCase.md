# Wireframe: 利用者（ディーラー/法人）新規案件作成

## 想定
MVPでは「詳細入力を全部はしない」方針。必須は最小限。

## レイアウト（フォーム）
```
┌─────────────────────────────────────┐
│ 上部ナビ：案件一覧 | 新規依頼 | 戻る   │
├─────────────────────────────────────┤
│ 新規依頼（必須項目のみ）               │
│                                         │
│ [車両/顧客情報：MVPでは最低限]             │
│ - 関連先名（顧客/依頼元）               │
│ - 車両情報（項目数を最小化）           │
│ - 依頼日                               │
│                                         │
│ [案件に紐づく“所定欄”初期化]            │
│ - 見積PDF欄（空で開始）               │
│ - 納品書PDF欄（空で開始）           │
│ - 途中写真欄（空で開始）            │
│                                         │
│ [プレビュー：案件IDが発行されます]     │
│ （作成後に案件詳細へ遷移）             │
│                                         │
│ [作成する]  [下書き保存（必要なら）]     │
└─────────────────────────────────────┘
```

## 注記（仕様）
- 作成時に `管理番号（案件ID）` を発行し、案件詳細ページへ遷移
- 以降の添付は `05_User_CaseDetail_Upload` で追加
