# アクションプラン（今すぐやること）

## 0. 前提（このアクションプランでのゴール）
- MVPを「見積を取れる粒度」まで固める
- 開発会社に渡すと、初期開発費・ランニング費・期間が見積可能になる状態にする
- デモ（案件1件分の登録〜閲覧まで）を短期間で作れる状態にする

## 1. 今日〜48時間（最優先：意思決定を確定）
1. 画像ストレージ方針を仮決め（MVP優先）
   - `自前サーバー`で進める/まずは`Google Drive`を検討する、どちらで見積依頼を出すか
   - 05_Storage_Cost_Comparison.mdの「次に確認すべき数値」を埋める（枚数/サイズ/保存期間/参照頻度）
2. MVPスコープを“追加しない”ルールで確定
   - MVPでは「案件作成」「添付（写真/PDF）」「案件詳細閲覧」「ステータス表示/更新（主に管理者）」に絞る
3. 権限の最小要件を最終確定
   - 利用者が自社案件のみ閲覧/操作できる（organization_idベース）

## 2. 2〜5日目（見積依頼・開発会社向けに固める）
1. ワイヤーフレームレビューを実施し、画面要件を最終化
   - `Documentation/Wireframes/` に作成した画面に対して「必須/不要」を確定
2. 開発会社へ見積依頼を送るための“仕様パック”を作る
   - 必ず添付するもの：`01_Platform_Proposal.md` / `02_MVP_PRD.md` / `06_Dev_Estimation_Spec.md` / `Wireframes/*`
3. 見積で聞くべき追加質問を統一（06_Dev_Estimation_Spec.mdに追記が必要なら反映）
   - 画像保存形式、バックアップ方針、権限実装方式、テスト/受入条件

## 3. 1週間目（DX補助金の要件確認と並行）
1. DX補助金の窓口（または担当者）へ質問
   - `07_DX_Subsidy_Checklist.md`の「最優先項目」をそのまま質問する
2. 回答を反映して、必要なら企画書（01）またはPRD（02）の文言・前提を調整

## 4. 2週間目（MVPの“デモ成立”を確認）
1. 開発会社からの見積案とリスクを比較
   - 期間差/ランニング差の理由（ストレージ・権限・品質担保）を整理
2. デモシナリオを1本固定
   - 「案件1件：新規作成 -> 写真/見積PDFアップロード -> 管理者が閲覧/ステータス -> 完了状態」までをデモ化
3. MVPの受入基準を最終化（02_の acceptance criteria を調整）

## 5. 成果物（このプラン完了の定義）
- ワイヤーフレームが全画面揃っている
- 開発会社へ送る仕様パックが完成している
- 補助金要件が“対象外/対象の可能性あり”いずれか判断できる
- ストレージ方針と概算のランニング目標（月3〜5万円）が見積に反映される
