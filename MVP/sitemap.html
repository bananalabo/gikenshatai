1. このMVPで解決したいこと（要点）

法人取引のやり取り（依頼・写真・見積/納品書・進捗）を `LINE/メール/電話` から 「案件ID（管理番号）」単位で一元化し、情報の転送・確認・漏れを減らします。

2. 利用者ロール（何が違うか）
Admin（元請け）

案件の閲覧・ステータス更新・必要な証憑（PDF/写真）の確認

User（ディーラー/法人）

自社案件の作成・見積/納品書のPDFアップロード・途中写真の追加

3. サイドメニュー（典型的なSaaS導線）
Admin

案件一覧 / インボックス

案件詳細は1ページで共通。Adminだけ「ステータス更新」を表示。

User

案件一覧 / 新規依頼 / インボックス

案件詳細は1ページで共通。Userだけ「PDFアップロード/写真追加」を表示。

4. 作成する画面（MVPスコープ）
画面 説明 プロトタイプ
入口 Admin/Userのどちらで見せるかを選ぶページ 開く
案件一覧 案件ID・関連先・ステータスで一覧。クリックで案件詳細へ Admin User
取引先 取引先（ディーラー/法人）単位で案件を分けて探しやすくする Admin User
新規依頼 Userが案件を作成（MVPでは必須項目のみ） 開く
案件詳細（共通1ページ） 同じ画面レイアウトで、roleに応じて「操作できる領域」が変わる Admin User
添付プレビュー 途中写真の拡大プレビュー / PDF表示（プロトタイプ） 開く
インボックス（一覧） 案件に紐づくメッセージを一覧（未読/既読） 開く
メッセージ（スレッド） 会話の流れを確認し、返信できるUI 開く

注：本ページ・画面群はバックエンドなしのUI/導線プロトタイプです。開発時は実データ連携を追加します。
