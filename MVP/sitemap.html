1. このMVPで解決したいこと（要点）

法人取引のやり取り（依頼・写真・見積/納品書・進捗）を LINE/メール/電話 から 「案件ID（管理番号）」単位で一元化し、情報の転送・確認・漏れを減らします。

2. 利用者ロール（何が違うか）
Admin（元請け）

案件の閲覧・ステータス更新・必要な証憑（PDF/写真）の確認

User（ディーラー/法人）

自社案件の作成・見積/納品書のPDFアップロード・途中写真の追加

3. ナビゲーション構造
Admin

案件一覧 / 取引先 / 受信箱

案件詳細は1ページで共通。Adminだけ「ステータス更新」を表示。

User

案件一覧 / 新規依頼 / 受信箱 / 取引先

案件詳細は1ページで共通。Userだけ「PDFアップロード/写真追加」を表示。

4. 作成する画面（MVPスコープ）
画面 説明 プロトタイプ
入口（トップ） Admin/Userのどちらで進むかを選ぶページ 開く
利用者ログイン 利用者向けログインページ（メール + パスワード） 開く
利用者新規登録 会社情報・担当者情報・利用規約同意 開く
管理者ログイン Admin専用ログインページ 開く
案件一覧 案件ID・関連先・ステータスで一覧。クリックで案件詳細へ Admin User
取引先 取引先（ディーラー/法人）単位で案件を分けて探しやすくする Admin User
新規依頼 Userが案件を作成（MVPでは必須項目のみ） 開く
案件詳細（共通） 同じ画面でroleに応じて「操作できる領域」が変わる Admin User
受信箱 案件に紐づくメッセージを一覧（未読/既読） 開く
メッセージスレッド 会話の流れを確認し、返信できるUI 開く
添付プレビュー 途中写真の拡大プレビュー / PDF表示 開く

注：本ページ・画面群はバックエンドなしのUI/導線プロトタイプです。

5. 次に確定したいこと（開発前）
  • 写真/添付の保存方式（自前サーバー vs Google Drive）
  • roleごとの入力/閲覧ルール（誤添付・漏れ防止）
  • MVPデモシナリオ（案件1件の登録→添付→閲覧→進捗更新）
6. 3分デモの推奨シナリオ
  1. 入口で「役割ごとに同じ案件を追跡できる」ことを説明
  2. Userで新規依頼を作成し、案件詳細で写真/書類を追加
  3. Adminで同案件を確認し、ステータスを更新
  4. Inboxで依頼・確認の往復を見せて、情報の一元化価値で締める
